VERSION CONTROL SYSTEM

Summary

Version Control System, disebut juga revision control system atau source code management adalah sistem yang mengelola perubahan dari sebuah dokumen program computer, website dan kumpulan informasi lain. Karena sistem ini merekam perubahan yang terjadi pada sebuah dokumen atau sekumpulan dokumen dari waktu ke waktu maka dimungkinkan untuk kembali ke versi tertentu di kemudian hari.

Jenis version control

Local Version Control System

basis data lokal yang terletak di local computer, yang merupakan tempat setiap perubahan file disimpan sebagai patch. Setiap kumpulan patch hanya berisi perubahan yang dibuat pada file dari versi terakhirnya.

Centralized Version Control System

Sistem ini memiliki sebuah server tunggal yang berisi seluruh versi dari dokumen, dan sejumlah orang yang mengunduh dokumen dari tersebut.

Distributed Version Control System

Dalam DVCS (seperti Git, Mercurial, atau Darcs), klien tidak hanya mengunduh salinan terbaru dari dokumen; melainkan, menyalin seluruh repositori, termasuk riwayat lengkapnya. Oleh karena itu, jika server mati dan sistem-sistem ini berkolaborasi melalui server tersebut, repositori klien mana pun dapat disalin kembali ke server untuk memulihkannya..
